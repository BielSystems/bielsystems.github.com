Segunda-feira

  • Idéia do cliente O cliente e o time conversam sobre o projeto visão de curto e longo prazo.
  • Desenho das telas O cliente e o time desenham como imaginam o projeto que será desenvolvido.

  • Priorização das tarefas O cliente prioriza as tarefas que serão realizadas durante a semana.

Terça e Quarta-feira

  • Desenvolvimento O time trabalha nas tarefas priorizadas pelo cliente.
  • Ambiente de Testes O projeto que está sendo desenvolvido fica disponivel para o cliente em um ambiente de testes.
  • Boas práticas O time utiliza de testes automatizados e todas as demais práticas do XP.

Quinta-feira

  • Finalização do desenvolvimento O time finaliza as tarefas da iteração.
  • Roteiro de testes No final do dia é entregue o roteiro de testes para o cliente.
  • Testeando o sistema O cliente tem de sexta a segunda para testar o que foi desenvolvido e trazer suas observações para próxima reunião.

Sexta-feira

  • Projetos internos Desenvolvimento dos projetos internos da HE:labs.
  • Open source O time contribui para projetos open source.
  • Cerveja e Red Bull É livre o consumo de cerveja, red bull e refrigerante para o time.
